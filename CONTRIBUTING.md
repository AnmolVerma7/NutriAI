# Contributing to NutriAI

## Commit Message Convention

We follow a strict commit message format to ensure history is readable and semantic.

**Format:** `TYPE: Description`

### Types

- **FEAT**: A new feature (e.g., `FEAT: Add login page`)
- **FIX**: A bug fix (e.g., `FIX: Resolve sidebar crash`)
- **TASK**: Chore or maintenance task (e.g., `TASK: Update dependencies`)
- **DOCS**: Documentation changes (e.g., `DOCS: Update README`)
- **STYLE**: Formatting, missing semi colons, etc; no code change
- **REFACTOR**: Refactoring production code

### Example

```bash
git commit -m "FEAT: Implement Supabase authentication"
```
